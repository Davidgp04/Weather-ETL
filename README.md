# CSV to PostgreSQL ETL Pipeline 🚀

This project implements an **ETL pipeline** using Python and Pandas to extract raw data from CSV files, clean and transform the dataset, and load the processed data into a PostgreSQL database.

The pipeline stages the cleaned data in a **PostgreSQL staging table**, which is then merged into a **production table** to maintain an updated and validated dataset.

The workflow is **orchestrated with Apache Airflow** and runs in a **containerized environment using Docker and Docker Compose**, enabling reproducibility and simplified deployment.

Intermediate data between pipeline stages is stored using **Parquet files** to improve I/O performance and maintain schema consistency.

# 🔄 Pipeline Workflow

## 1️⃣ Extraction

The pipeline extracts raw weather data from CSV files using **Pandas**.

The extracted dataset is converted into a **Parquet file**, which improves input/output performance and ensures a consistent data schema between pipeline stages.

---

## 2️⃣ Transformation

During the transformation stage, the pipeline reads the Parquet file produced during extraction and performs several data-cleaning operations:

- Removes rows containing null values  
- Removes duplicate records  
- Normalizes the `date_time` column  
- Filters out invalid or impossible values  
- Aggregates rows that share the same `location` and `date_time`  

The aggregation step computes an **average of the measurements** for duplicate observations to produce a single consolidated record.

The cleaned dataset is then written to a new **Parquet file**, which serves as the input for the loading stage.

---

## 3️⃣ Load

The load stage reads the cleaned Parquet dataset and inserts the records into a **PostgreSQL staging table**.

Once the data is staged, a **merge operation** is executed to synchronize the staging table with the production table.

This step:

- Inserts new records  
- Updates existing records when conflicts occur  

To ensure data quality, the **production table includes several constraints**, such as value checks and uniqueness rules.

---

# Airflow Orchestration

The ETL pipeline is orchestrated using **Apache Airflow** through a Directed Acyclic Graph (DAG).

The DAG contains three tasks:

### 1️⃣ Extract

- Reads CSV files from the raw data directory  
- Converts the data into a Pandas DataFrame  
- Writes the dataset to a Parquet file in `data/processed`

### 2️⃣ Transform

- Reads the Parquet dataset generated by the extraction task  
- Cleans invalid records, duplicates, and null values  
- Aggregates duplicate observations  
- Writes the cleaned dataset to a new Parquet file in `data/processed`

### 3️⃣ Load

- Establishes a connection to the PostgreSQL database  
- Reads the cleaned Parquet dataset  
- Loads the data into a staging table  
- Executes a merge query to update the production table

# ⚙️ Setup & Installation

## 📋 Prerequisites

- Docker  
- Docker Compose  

---

## ▶️ Running the Project

```
cp .env.example .env
echo "AIRFLOW_UID=$(id -u)" >> .env
docker compose up --build
```

This command initializes the Docker environment and starts all required services, including **Airflow and PostgreSQL**.

---

## 🌐 Access Airflow UI

Airflow automatically generates login credentials when the environment starts.

To retrieve them, run:

```
sudo docker compose logs airflow | grep "user"
```

Once you obtain the credentials, open:

```
http://localhost:8080
```

If running on a cloud instance, access the interface using:

```
http://<PUBLIC-IP>:8080
```

Make sure **port 8080 is open in your firewall settings**.

# 🗄 Database Schema

## Production Table

| Column | Data Type | Constraints | Description |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | Unique auto-incrementing identifier |
| location | TEXT | UNIQUE* | Name of the weather station or city |
| date_time | TIMESTAMP | UNIQUE* | Timestamp of the observation |
| temperature_c | REAL | CHECK (-50 to 60) | Temperature in Celsius |
| humidity_pct | REAL | CHECK (0 to 100) | Humidity percentage |
| precipitation_mm | REAL | CHECK (>= 0) | Rainfall in millimeters |
| wind_speed_kmh | REAL | CHECK (>= 0) | Wind speed in km/h |
| sample_count | INTEGER | NOT NULL, DEFAULT 1 | Number of aggregated samples |

\* `location` and `date_time` together represent a **unique observation key**.

---

## Staging Table

| Column | Data Type | Description |
|------|------|------|
| location | TEXT | Raw location identifier |
| date_time | TIMESTAMP | Raw observation timestamp |
| temperature_c | REAL | Raw temperature measurement |
| humidity_pct | REAL | Raw humidity measurement |
| precipitation_mm | REAL | Raw precipitation measurement |
| wind_speed_kmh | REAL | Raw wind speed measurement |

The staging table acts as a **temporary validation layer before updating the production dataset**.
